Sensor Data Collection Framework

1. Description

Android-based platform for collection of sensor data from a consumer smartwatch. The developed prototype is capable of performing real-time cloud synchronisation of data acquired from the three onboard sensors at the sampling rate of 100 Hz. The distinguishing feature of the framework lies in its ability to handle wireless connectivity loss in order to preserve temporal data integrity while maintaining uninterrupted operation.

2. System Architecture

The system consists of an Android smartwatch, tablet, and server. The wearable utilises Bluetooth Low Energy (BLE) connection to the handheld for data interchange. The handheld maintains a connection to the cloud server over the Internet.

The wearable application acquires data from the hardware sensors - accelerometer, gyroscope, magentometer - and transmits sensor data to the companion mobile device. The mobile application visualises sensor values and relays data to the cloud storage over HTTP. The web application on the dedicated server stores data to the database and provides API for data retrieval.

The system ensures temporal continuity of data through offline synchronisation via temporary data storage buffers. In case of connectivity loss between the nodes, the data is buffered and synchronised when the connection is re-established. The mechanism ensures that network downtime, unstable connection, and data loss during transmission do not result in data loss and hence affect data consistency.

4. Requirements

  • Android Wear 2.0 Smartwatch
  • Android Mobile Device

    • The devices must be compatible for pairing and data exhange over Bluetooth.

  • SymmetricDS Andorid Service

    • The JAR files can be downloaded from: https://sourceforge.net/projects/symmetricds/files/symmetricds/symmetricds-3.8/symmetric-android-3.8.42.zip/download

5. Installation

  1. Pair the smartwatch with the handheld.
  2. Enable Developer Options and activate debugging mode on both devices.
  3. Clone Git repository from https://github.com/cidcom/gleb-android-imu-platform
  4. Download SymmetricDS dependency files and import into the mobile module under /libs directory.
  5. Build and install the wearable and mobile apps on the respective devices
  6. Start data retreival from the main screen of the mobile device.
